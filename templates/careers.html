01 · Why join Sohana

Four reasons that should matter to you.

Real-world relevance

Work on a product whose informal version is already used by hundreds of millions of people. We're not creating demand — we're meeting it.

Build systems that matter

Help build the financial coordination layer that improves access to capital for the people who have always built it themselves — without a formal system.

Be on the foundational team

Join before structure hardens. Decisions you make in the next 18 months will compound for the next 10 years. This phase doesn't come twice.

Designed for long-term scale

Sohana isn't a feature. It's infrastructure designed to compound across Africa and global diaspora markets over the next decade. The work has reach.

FE
Frontend & Mobile Engineer
Engineering · Web + Mobile · Remote-first
The work

Build clean, fast, mobile-responsive interfaces for the Sohana platform — dashboards, wallet flows, ROSCA management, and the consumer-facing mobile app. You'll own front-end architecture decisions and ship features end-to-end.

Key requirements
  • Strong React proficiency — and bonus for React Native or Flutter for the mobile track
  • Solid UI/UX intuition — you can take a Figma file and ship something better than the spec
  • Mobile-responsive web — comfortable with performance on mid-range Android over 3G
  • Bonus: experience with HTML/Jinja stacks (current MVP) or progressive web apps
What success looks like (first 6 months)

The mobile experience feels native. Page loads under 2 seconds on real-world African networks. You've shipped at least three flagship user flows and influenced how the product feels.

BE
Backend Engineer
Engineering · APIs · Database · Remote-first
The work

Own the core platform logic — APIs, ledger integrity, transaction flows, KYC pipeline. You'll architect for both correctness (this is financial software) and scale (multi-region, multi-currency).

Key requirements
  • Solid Python or Node.js — current stack is Python/Flask/SQLite, evolving to Postgres
  • Database design fluency — financial data, audit trails, idempotency
  • API security mindset — auth, signing, rate-limiting, replay protection
  • Bonus: experience with payments integrations (Stripe, mobile money, EMIs)
What success looks like (first 6 months)

Zero ledger discrepancies. APIs documented and stable. Payment-rail integration work is unblocked. You've made the system both more reliable and easier to extend.
